# ADR-0001: Industrial Dispatcher baseline

## Статус

Accepted

## Контекст

Репозиторий `dispatcherV4` использовался для учебного SCADA-lite прототипа. После подготовки продуктовой концепции, UX/UI-спецификации, Core UI prototype, master-ТЗ и AI-focused execution guide принято решение начать промышленную реализацию с чистого baseline.

Среда разработки, GitHub remote, локальный путь и рабочий процесс уже настроены. Создание нового репозитория увеличит организационные издержки без технической необходимости.

## Решение

Использовать существующий репозиторий, но очистить содержимое старого тренировочного проекта обычным commit без переписывания Git-истории.

После очистки репозиторий содержит:

- `README.md`;
- `PROJECT_STATE.md`;
- `DISPATCHER_TECHNICAL_SPECIFICATION_AND_ROADMAP.md`;
- `DISPATCHER_AI_IMPLEMENTATION_SPEC.md`;
- baseline repo settings;
- `docs/reference/` с исходными материалами и Core UI prototype;
- `docs/runbooks/`;
- `docs/adr/`.

Старые проекты `src/`, `tests/`, старые миграции, старые Blazor pages и учебные сущности `Device/Tag` не переносятся автоматически.

## Последствия

Плюсы:

- сохраняется настроенный GitHub remote;
- сохраняется Git history как журнал обучения;
- промышленная разработка начинается с чистого текущего состояния;
- AI-focused guide может использовать единый репозиторий и `PROJECT_STATE.md`.

Минусы:

- в истории Git остаются старые учебные коммиты;
- при поиске по GitHub history можно увидеть старую модель, которая больше не является архитектурным источником;
- нужно явно не переносить старые `Device/Tag` решения в новый продукт.

## Инварианты

- Master-ТЗ и AI guide являются источниками истины.
- Старый учебный код не считается production baseline.
- История Git не переписывается без отдельного решения.
- C++ не начинается в MVP.
- Реальные команды управления отключены до command safety gate.
